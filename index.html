آموزش کامل دستورات و مفاهیم SQL

از SELECT تا Window Functions، همه چیز را با توضیحات کامل و مثال‌های عملی یاد بگیر. روی هر مبحث کلیک کن تا جزئیات رو ببینی.

سطح مبتدی

۹ مفهوم

SELECT

انتخاب و نمایش داده‌ها از جدول

مبتدی

توضیحات

دستور SELECT برای دریافت و نمایش داده‌ها از یک یا چند جدول استفاده می‌شود. این پایه‌ترین و پرکاربردترین دستور در SQL است.

می‌توانید ستون‌های خاصی را انتخاب کنید
با * همه ستون‌ها را نمایش دهید
با AS نام ستون‌ها را تغییر دهید

مثال 

SELECT first_name, last_name, email
FROM students;
نام، نام‌خانوادگی و ایمیل همه دانشجویان را نمایش می‌دهد.
|

WHERE

فیلتر کردن داده‌ها با شرایط

مبتدی

توضیحات

دستور WHERE برای فیلتر کردن رکوردها بر اساس شرایط مشخص استفاده می‌شود. فقط رکوردهایی که شرط را برآورده کنند، نمایش داده می‌شوند.

عملگرهای مقایسه‌ای: =، >، <
عملگرهای منطقی: AND، OR، NOT

مثال 

SELECT first_name, last_name
FROM students
WHERE enrollment_year = 2025
AND gpa >= 17;
دانشجویان سال ۲۰۲۵ با معدل بالاتر از ۱۷.
|

COUNT

شمارش تعداد رکوردها

مبتدی

توضیحات

تابع COUNT() تعداد رکوردهایی را که با شرط مطابقت دارند، شمارش می‌کند. با COUNT(*) همه رکوردها و با COUNT(column) تعداد مقادیر غیر NULL را شمارش می‌کند.

همراه با GROUP BY برای دسته‌بندی استفاده می‌شود
مقادیر NULL را شمارش نمی‌کند

مثال 

SELECT COUNT(*) AS total_students
FROM students
WHERE status = 'active';
تعداد دانشجویان فعال را نمایش می‌دهد.
|

SUM

محاسبه مجموع مقادیر

مبتدی

توضیحات

تابع SUM() مجموع مقادیر یک ستون عددی را محاسبه می‌کند. معمولاً با GROUP BY برای محاسبه مجموع در هر گروه استفاده می‌شود.

فقط روی ستون‌های عددی قابل استفاده است
مقادیر NULL را نادیده می‌گیرد

مثال 

SELECT SUM(amount) AS total_sales
FROM orders
WHERE date = '2025-01-01';
مجموع فروش در تاریخ مشخص را محاسبه می‌کند.
|

AVG

محاسبه میانگین مقادیر

مبتدی

توضیحات

تابع AVG() میانگین مقادیر یک ستون عددی را محاسبه می‌کند. برای محاسبه میانگین نمرات، حقوق، قیمت و ... استفاده می‌شود.

فقط روی ستون‌های عددی قابل استفاده است
می‌توان با ROUND() گرد کرد

مثال 

SELECT AVG(gpa) AS average_gpa
FROM students
WHERE department = 'Computer Science';
میانگین معدل دانشجویان رشته کامپیوتر.
|

MIN

پیدا کردن کمترین مقدار

مبتدی

توضیحات

تابع MIN() کوچکترین مقدار را در یک ستون پیدا می‌کند. برای پیدا کردن کمترین قیمت، جوان‌ترین سن، کمترین نمره و ... استفاده می‌شود.

روی ستون‌های عددی و متنی قابل استفاده است
مقادیر NULL را نادیده می‌گیرد

مثال 

SELECT MIN(price) AS cheapest_product
FROM products;
ارزان‌ترین قیمت محصولات را پیدا می‌کند.
|

MAX

پیدا کردن بیشترین مقدار

مبتدی

توضیحات

تابع MAX() بزرگترین مقدار را در یک ستون پیدا می‌کند. برای پیدا کردن گران‌ترین قیمت، مسن‌ترین سن، بالاترین نمره و ... استفاده می‌شود.

روی ستون‌های عددی و متنی قابل استفاده است
مقادیر NULL را نادیده می‌گیرد

مثال 

SELECT MAX(salary) AS highest_salary
FROM employees;
بالاترین حقوق کارمندان را پیدا می‌کند.
|

DELETE

حذف رکوردها از جدول

مبتدی

توضیحات

دستور DELETE برای حذف رکوردها از یک جدول استفاده می‌شود. هشدار: بدون WHERE همه رکوردها حذف می‌شوند!

همیشه از WHERE استفاده کنید
قابل بازگشت نیست (مگر با ROLLBACK)

مثال 

DELETE FROM students
WHERE id = 123;
دانشجویی با شناسه ۱۲۳ را حذف می‌کند.
|

UPDATE

بروزرسانی رکوردها

مبتدی

توضیحات

دستور UPDATE برای تغییر مقادیر رکوردهای موجود در جدول استفاده می‌شود. توجه: بدون WHERE همه رکوردها تغییر می‌کنند!

همیشه از WHERE استفاده کنید
می‌توان چند ستون را همزمان تغییر داد

مثال 

UPDATE students
SET gpa = 18.5, status = 'graduated'
WHERE id = 456;
معدل و وضعیت دانشجو با شناسه ۴۵۶ را تغییر می‌دهد.
|

۳ مفهوم

RANDOM

انتخاب تصادفی رکوردها

متوسط

توضیحات

تابع RANDOM() (یا RAND() در برخی دیتابیس‌ها) برای انتخاب تصادفی رکوردها استفاده می‌شود. معمولاً با ORDER BY ترکیب می‌شود.

برای نمایش رکوردهای تصادفی مفید است
در هر بار اجرا نتیجه متفاوت است

مثال 

SELECT name, question
FROM quiz_questions
ORDER BY RANDOM()
LIMIT 5;
۵ سوال تصادفی از بانک سوالات را انتخاب می‌کند.
|

PRIMARY KEY

کلید اصلی جدول

متوسط

توضیحات

PRIMARY KEY (کلید اصلی) یک ستون یا ترکیبی از ستون‌ها است که به‌طور یکتا هر رکورد را در جدول مشخص می‌کند. ویژگی‌ها: یکتا (UNIQUE) و غیر NULL.

هر جدول فقط یک PRIMARY KEY دارد
معمولاً از AUTO_INCREMENT استفاده می‌شود

مثال 

CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTO_INCREMENT,
    first_name TEXT,
    last_name TEXT,
    email TEXT
);
ستون id به‌عنوان کلید اصلی، هر دانشجو را یکتا می‌کند.
|

FOREIGN KEY

کلید خارجی و ارتباط بین جداول

متوسط

توضیحات

FOREIGN KEY (کلید خارجی) ستونی است که به PRIMARY KEY جدول دیگر ارجاع می‌دهد. برای ایجاد ارتباط بین جداول و حفظ یکپارچگی داده‌ها استفاده می‌شود.

ارتباط one-to-many را پیاده‌سازی می‌کند
با ON DELETE CASCADE حذف زنجیره‌ای

مثال 

CREATE TABLE enrollments (
    id INTEGER PRIMARY KEY AUTO_INCREMENT,
    student_id INTEGER,
    course_id INTEGER,
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (course_id) REFERENCES courses(id)
);
جدول ثبت‌نام که به جداول students و courses متصل است.
|

مفاهیم پیشرفته

۲ مفهوم

One-to-One

رابطه یک به یک بین جداول

پیشرفته

توضیحات

رابطه One-to-One زمانی است که هر رکورد در جدول A دقیقاً به یک رکورد در جدول B مرتبط است و بالعکس. معمولاً برای جدا کردن داده‌های حساس یا داده‌های با تعداد ستون زیاد استفاده می‌شود.

مثال: کاربر و پروفایل
کلید خارجی معمولاً UNIQUE است

مثال 

CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    username TEXT
);

CREATE TABLE profiles (
    id INTEGER PRIMARY KEY,
    user_id INTEGER UNIQUE,
    bio TEXT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
هر کاربر دقیقاً یک پروفایل دارد.
|

One-to-Many

رابطه یک به چند بین جداول

پیشرفته

توضیحات

رابطه One-to-Many (یک به چند) رایج‌ترین نوع رابطه در دیتابیس‌های رابط‌ای است. هر رکورد در جدول A می‌تواند به چندین رکورد در جدول B مرتبط باشد، اما هر رکورد در B فقط به یک رکورد در A متصل است.

مثال: دانشجو و ثبت‌نام‌ها
کلید خارجی در جدول Many قرار می‌گیرد

مثال 

CREATE TABLE authors (
    id INTEGER PRIMARY KEY,
    name TEXT
);

CREATE TABLE books (
    id INTEGER PRIMARY KEY,
    title TEXT,
    author_id INTEGER,
    FOREIGN KEY (author_id) REFERENCES authors(id)
);
هر نویسنده می‌تواند چندین کتاب داشته باشد.
